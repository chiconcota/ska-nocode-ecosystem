## 2024-06-09 - Accessible Icon Buttons in Editor Components
**Learning:** When using `<span role="button">` or `<div>` for interactive elements like the Tailwind classes copy and preset save buttons in custom Gutenberg blocks or plugins, they often lack native keyboard focus (`tabIndex={0}`), ARIA labels, and keyboard event handlers (`onKeyDown` for Enter/Space), rendering them inaccessible to keyboard and screen reader users. Additionally, decorative inner elements like `material-symbols-outlined` spans or SVG icons need `aria-hidden="true"` and `focusable="false"` to prevent redundant screen reader announcements and focus issues.
**Action:** Always verify that custom interactive elements acting as buttons have `tabIndex={0}`, an appropriate `aria-label`, an `onKeyDown` handler that responds to Enter and Space keys, and that any inner decorative icons are explicitly hidden from assistive technologies.
