## 2024-06-30 - Fix Interactive Span Elements Accessibility
**Learning:** In the ska-no-code-design plugin's Tailwind panel components, several interactive controls are built using `span` elements with `role="button"`. However, they lack keyboard accessibility (no `tabIndex` or keyboard event handlers) and screen-reader accessibility for conditionally disabled states (missing `aria-disabled`). Furthermore, Material Symbols and SVG icons inside these spans are not hidden, potentially causing screen reader confusion.
**Action:** When creating or modifying custom button elements (e.g., `<span role="button">`), always include `tabIndex={0}` (or conditionally `tabIndex={-1}` when disabled), an `onKeyDown` handler for 'Enter' and 'Space' keys, `aria-disabled` if conditionally active, `aria-label` for screen readers, and set `aria-hidden="true"` and `focusable="false"` on internal icons to ensure full accessibility.
