# [ARCHITECTURAL PILLARS - BÀN GIAO PHASE 1, 2, 3 & 4]
> Đây là các nguyên tắc kiến trúc cốt lõi hiện hành (Current Primitives) đúc kết và kế thừa qua cả 4 Phase phát triển. Chúng đóng vai trò giữ hệ thống ổn định và ngăn AI phá vỡ quy tắc, nhưng HOÀN TOÀN CÓ THỂ PIVOT (đập đi xây lại/thay đổi) bất cứ lúc nào nếu tầm nhìn và trải nghiệm thực tế của bạn lên tiếng. Quyền định đoạt tối thượng luôn thuộc về bạn!

- **1. Kiến Trúc "The Core Four" & Nền Tảng Agnostic:** Chuyển dịch từ Page Builder sang App Builder Framework. Sử dụng WordPress làm máy chủ tạm thời (Authentication & Admin UI). Hệ thống chia rẽ 4 giới hạn (Decoupled Boundaries): Ska Theme (Zero CSS), Ska No-code Design (Thiết kế & Biên dịch Tailwind JIT), Ska Data Pro (Dữ liệu), và Ska Logic Engine (Luồng sự kiện). Giao tiếp chéo qua do_action và apply_filters, cực hạn việc gọi Class trực tiếp.
- **2. The Trinity Data Model (Ska Data Pro):** Khai tử triệt để mô hình EAV (wp_postmeta) lề mề của WordPress. Mọi Smart Object/App phải được khởi tạo bằng Bảng Phẳng (Flat Tables: ska_data_*) với Native JSON type cho multi-select/arrays. Blueprint (JSON) định tuyến cấu trúc bằng Relative Slugs (Ký danh nội bộ) và Auto-grouping để xử lý Collision an toàn.
- **3. Cỗ Máy Tailwind JIT CSS & Clean Slate:** Server-side compiler với khả năng parse linh hoạt từ Tailwind v3 đến V4 pseudo-classes (group-has-checked, [-offset]). Tuyệt đối bảo vệ **Clean Slate** (Mâm xôi thuần khiết): Không dùng !important tùy tiện, dọn sạch CSS rác của WordPress bằng Specificity Scope (:where(), body .ska-builder), trả toàn quyền định dạng class cho người dùng tạo Atomic Component.
- **4. SkaFX DSL & Universal Binding (Ska Logic Engine):** Thống nhất Data Fetch và Logic Rule vào 1 cỗ máy biểu thức duy nhất (SkaFX). Bất kể giao diện nào cũng hỗ trợ Nội suy dữ liệu {{...}}. Cache toàn bộ Cấu trúc Bảng ska_data_dictionary vào RAM tĩnh của PHP (thời gian quét Suffix <1ms) để hỗ trợ Định danh ngữ cảnh siêu mượt mà.
- **5. Native Backend Integration:** Hệ thống quản lý của người dùng (App Portals) sử dụng Chung giao diện Unified Canvas với thẻ Tailwind, nhưng bảo mật qua cờ publicly_queryable = false. Bất cứ API tương tác nào từ Frontend đều trả về dữ liệu bảo vệ kỹ lưỡng bằng Nonce và Data Healing (Cứu thương mảng Array bị lỗi).
- **6. Dedicated Page & Early Intercept (Ska App Router):** Thay vì kiến trúc SPA (Single Page Application) gây phình to DOM, hệ thống chuyển hẳn sang mô hình Dedicated Page định tuyến theo URL động. Đánh chặn (Intercept) luồng render sớm tại hook `parse_request` để nạp template và kiểm tra quyền truy cập (Role Checking) trước khi tải lõi WordPress dư thừa.
- **7. Shadow Scratchpad & isolated Editor:** Thực hiện thiết kế Rich Text ngoài Frontend qua môi trường Iframe cô lập của CPT ảo (`ska_scratchpad`), đồng bộ trực tiếp với form state qua JS Bridge và xóa bỏ CPT nháp ngay lập tức khi tắt modal để giữ database sạch sẽ.
- **8. Macro Pattern Injector (Atomic Preservation):** Thiết lập việc tự động tạo view bằng cách rải các khối Atomic (Ska Loop, Ska Text, Ska Button, Ska Modal) đã cấu hình sẵn Event, thay vì dùng các khối đóng hộp (Blackbox block) để bảo vệ tuyệt đối quyền tuỳ biến tự do (FSE) của Power User.

## 2026-05-30 - 🟢 Hoàn thành: Vá lỗi block select liên kết động trên cả Editor và Frontend (Ska No-Code Design v1.0.3 & Ska Logic Engine v1.1.2)

Hệ thống sẽ tự động thay thế thuộc tính `value` và nhãn `<span>` động tương ứng với dữ liệu CSDL.
- **Decision (Form Init Lifecycle Race Condition Fix):** Gỡ bỏ lệnh trì hoãn `$nextTick` trong hàm `init()` của `skaForm` (file `ska-frontend.js`) để khởi tạo các thuộc tính form đồng bộ trước khi Alpine biên dịch DOM. Điều này ngăn chặn race condition khiến các checkbox liên kết động bị Alpine coi là boolean checkbox và xóa sạch giá trị `value` trên DOM.
- **Decision (JSX i18n Curly Braces Fix):** Bọc các hàm i18n `__()` trong dấu ngoặc nhọn `{}` cho các thuộc tính component (như `placeholder`, `help`, `label`, `instructions`) phân tán trên 10 file nguồn JSX để sửa triệt để các lỗi biên dịch của Webpack.

## 2026-05-30 - 🟢 Hoàn thành: Khắc phục lỗi lưu CSDL JSON khi đệ trình mảng rỗng hoặc CSV thô (Ska Data Pro v1.0.3 & Ska Logic Engine v1.1.1)

- **Decision (Universal JSON Parse Fallback):** Cập nhật hàm `Data_Fetcher::enrich_relations()` hỗ trợ giải mã cả định dạng JSON Array chứa các ID số nguyên/đối tượng lẫn chuỗi CSV thô để bảo toàn tính tương thích ngược cho dữ liệu cũ.
